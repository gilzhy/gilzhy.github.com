Android学习笔记-Broadcast

说明: 内容取自中山大学软件学院刘宁老师的SE-806课程的课件

之前说过Intent是一个对动作和行为的抽象描述,负责组件之间与程序之间进行消息传递

而Broadcast Receiver组件把Intent作为一个消息广播出去,由所有对其感兴趣的程序对其作出反应.

Android广播机制包含三个基本要素:

  • Broadcast广播 —发送广播,广泛运用在应用程序之间传输信息
  • BroadcastReceiver广播接收器 —接受广播,对发送出来的Broadcast进行过滤接受并响应
  • Intent意图 —保存广播相关信息

广播过程描述

  1. 首先在需要发送信息的地方,把要发送的信息和用于过滤的信息(如Action、Category)装入一个Intent对象,然后通过调用Context.sendBroadcast() 方法,把Intent对象以广播方式发送出去。
  2. 当Intent发送以后,所有已经注册的BroadcastReceiver会检查注册时的IntentFilter是
    否与发送的Intent相匹配,若匹配则就会调用BroadcastReceiver的void onReceive(ContextcurContext, Intent broadcastMsg)方法。

Broadcast机制

之前说过通过intent实现Activity间跳转的两个机制:

  • 显示Intent,构造Intent对象时就指定接收者
  • 隐式Intent,发送者构造Intent对象时并不知道也不关心接收者是谁,只通过filter来选择接收者(有利于降低发送者和接收者之间的耦合)

Broadcast的实现机制与隐式Intent类似

Broadcast生命周期

  • 当系统通过filter字符串找到匹配的Receiver, onReceive方法就会被执行。但是onReceive方法必须在5秒内执行完毕退出,否则会导致FC(Force Close强制关闭)
  • 通常来说,BroadcastReceiver用来更新content、启动service、更新UI或者通过notification manager在状态栏中提示。5秒的限制保证了主要处理任务不会、也不应该在BroadcastReceiver中完成。
  • 如果需要执行大量任务,可以在BroadcastReceiver中启用Service。

Broadcast手机自动服务

前面我们介绍了如何实现一个自定义的Broadcast,但是Broadcast的应用绝对不仅这些。
在Android操作系统中,有许多与手机相关的事件会对系统发送广播信息。当系统发出广播后,就会搜索是否注册了负责处理该广播的BroadcastReceiver。
因此,只要我们了解系统的广播类型,我们就可以实现很多手机自动服务功能,例如,收到新短信进行提示、手机来电时自动拒接等等。

Linux下Android真机调试

问题

Run as Android Application后显示的手机设备是unkown,名字是?????,即手机无法被识别

解决

使用Android SDK提供的adb工具(原先的SDK在tools里,现在已经移到platform-tools了)

进入platform-tools,运行adb devices:

zhyin@zhyin-K42Jc:~/adt-bundle-linux-x86-20130522/sdk/platform-tools$ ./adb devices

得到的内容是:???????????? no permissions

原因是启动adb时要root权限,重启adb加sudo即可:

adb kill-server

sudo adb start-server

在adb devices即可看到自己的手机已经被识别了

Android学习笔记-事件处理

说明: 内容取自中山大学软件学院刘宁老师的SE-806课程的课件

Bundle类型

Bundle是个类型安全的容器,它的实现其实是对HashMap做了一层封装。对于HashMap来讲,任何名值对都可以存进去,值可以是任何的Java对象。
但是对于Bundle不是这样。同样存的是名值对,但是这个值只能是基本类型或基本类型数组。如:String、int、byte、boolean、char等。

Framework构成

主要四部分:

  • Activities 管理应用程序展示
    • Activity Life Cycle
  • Services 管理后台服务
    • Services Life Cycle
  • Broadcast receivers 管理事件的广播与接收
  • Content Provider 管理数据共享
    • Android系统中,数据是私有的,包含文件数据和数据库数据。Content Provider类提供了标准方法接口,让其他的应用程序读取。必须在AndroidManifest.xml中声明

Intent(一次即将操作的抽象描述,各部分沟通的桥梁)

Intent是一种运行时绑定机制,他能在应用程序运行的过程中连接两个不同的组件,实现组件间的跳转。

举例:不同Activity之间的通讯,属于不同UI线程之间的通讯,如果要在不同的Activity之间传递消息,则需要用到Intent。在Intent中告诉操作系统要选择哪个目标Activity进行实例化,并传递消息

Activity、Service和BroadcastReceiver,都是通过Intent机制激活的,而不同类型的组件在传递Intent时有不同的方式。

Android 使用了Intent 这个特殊类,实现在屏幕与屏幕之间移动
Intent 是描述应用想要做什么,其组成是:

  • 动作(Action):描述Intent所触发动作名字的字符串.典型的动作类型有:MAIN(活动的门户)、VIEW、PICK、EDIT等。
  • 动作对应的数据(Data):描述Intent要操作的数据URI和数据类型
  • 组件名称: Intent目标组件的名称。
  • Category: 对被请求组件的额外描述信息。
  • Extra: 当我们使用Intent连接不同的组件时,有时需要在Intent添加额外的信息,以便把数据传递给目标Activity。
  • Flag: 借助 Intent 中的 flag 和 AndroidMainifest.xml 中activity 元素的属性,就可以控制到 Task 里 Activity 的关联关系和行为。

例如:要查看某个人的联系方式,你需要创建一个动作类型为VIEW 的Intent,以及一个表示这个人的URI Intent intent = new Intent(Intent.ACTION_VIEW, uri)

Intent激活方式

  • Activity: 通过调用Context.startActivity()或者Activity.startActivityForResult()方法。
  • Service: 调用Context.startService()或者Context.bindService()方法将调用这方法的上下文与Service绑定。
  • BroadcastReceiver: 通过Context.sendBroadcast()、Context.sendOrderBroadcast()和Context.sendStick
  • Broadcast()发送BroadcastIntent。 BroadcastIntent发送后,所有已注册的拥有与之匹配的IntentFilter的BroadcastIntent就会被激活。

IntentFilter

  • intentfilter 则用于描述一个activity(或者 IntentReceiver)能够操作哪些intent。
    • 一个activity 如果要显示一个人的联系方式时,需要声明一个IntentFilter,这个IntentFilter 要知道怎么去处理VIEW 动作和表示一个人的URI。

  • IntentFilter 需要在AndroidManifest.xml中定义

  • 通过解析各种intent,从一个屏幕导航到另一个屏幕是很简单的。

    • 当向前导航时,activity 将会调用startActivity(IntentmyIntent)方法。
    • 然后,系统会在所有安装的应用程序中定义的IntentFilter中查找,找到最匹配Intent 对应的activity。
    • 新的activity 接收到通知后,开始运行。
    • 当startActivity 方法被调用将触发解析Intent 的动作。
  • 这个机制提供了两个关键好处:
    • Activities 能够重复利用从其它组件中以Intent 的形式产生的一个请求;
    • Activities 可以在任何时候被一个具有相同IntentFilter 的新的Activity 取代。

IntentReceiver

  • IntentReceiver 在AndroidManifest.xml 中注册,也可在代码中使用Context.registerReceiver()进行注册。
  • 当一个IntentReceiver 被触发时,应用不必对请求调用IntentReceiver,系统会在需要的时候启动你的应用。
  • 各种应用还可以通过使用Context.broadcastIntent()将自己的IntentReceiver 广播给其它应用程序。
  • 当希望应用能够对一个外部的事件(如电话呼入,数据网络可用,或者某个定时)做出响应,可以使用一个IntentReceiver。虽然IntentReceiver 在感兴趣的事件发生时,会使用NotificationManager通知用户,但它并不能生成一个UI。

Intent的调用方式

  • 显式Intent: 明确指出了目标组件名称。显式Intent更多用于在应用程序内部传递消息。比如在某应用程序内,一个Activity启动一个Service。
  • 隐式Intent: 没有明确指出了目标组件名称。 Android系统使用IntentFilter来寻找与隐式Intent相关的对象。隐式Intent恰恰相反,它不会用组件名称定义需要激活的目标组件,它更广泛地用于在不同应用程序之间传递消息

事件监听

  • OnClickListener public void onClick(View v) 点击
    • v: 事件源控件
  • OnLongClickListener public boolean onLongClick(View v) 长按
    • 返回true时: 表示已经完整地处理了这个事件,并不希望其他的回调方法再次进行处理
    • 返回false时: 表示并没有完成处理该事件,更希望其他方法继续对其进行处理
  • OnFocusChangeListener Public void onFocusChangeListener(View V, Boolean hasFocus) 焦点变化
    • 参数hasFocus 表示v的新状态,即v是否获得焦点
  • OnKeyListener public Boolean onkey(View v,int keyCode, KeyEvent event) 键盘监听
    • keyCode: 手机键盘的键码
    • event: 键盘事件封装类的对象,包含了事件的详细信息
  • OnTouchListener Public boolean onTouch(View V ,MotionEvent event) 手机屏幕事件(按下,抬起,滑下等)
    • event: 事件封装类的对象,封装了触发事件的详细信息(事件类型,触发事件等)

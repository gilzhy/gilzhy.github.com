SublimeText2快捷键

  • esc后用vim快捷建
  • Alt+/ 补全
  • Ctrl+D 选词
    • 反复按快捷键，即可继续向下同时选中下一个相同的文本进行同时编辑
    • Alt+F3 选中文本按下快捷键，即可一次性选择全部的相同文本进行同时编辑
  • Ctrl+P 输入@相当于Ctrl+R 输入 : 相当于Ctrl+G
    • Ctrl+G 跳转到相应的行
    • Ctrl+R 查找文件主标题/函数
  • 插入空行
    • Ctrl+Enter 插入行后（快速换行）
    • Ctrl+Shift+Enter 光标前插入行
  • 书签
    • Ctrl+F2 设置书签
    • F2 下一个书签
    • Shift+F2 上一个书签
  • 注释
    • Ctrl+Shift+/ 注释已选择内容
    • Ctrl+/ 注释整行（如已选择内容，同Ctrl+Shift+/）
  • Ctrl+Y 恢复撤销
  • Alt+Shift+1~9 多窗口
  • F6 检测语法错误
  • F9 行排序(按a-z)
  • F11 全屏模式键盘
  • Ctrl+Shift+P 打开命令面板
  • Ctrl+J 合并行（已选择需要合并的多行时）
  • Ctrl+L 选择整行（按住-继续选择下行）
  • Ctrl+M 光标移动至括号内开始或结束的位置
  • Ctrl+K Backspace 从光标处删除至行首
  • Ctrl+KK 从光标处删除至行尾
  • Ctrl+Shift+A 选择光标位置父标签对儿
  • Ctrl+Shift+L 鼠标选中多行（按下快捷键），即可同时编辑这些行
  • Ctrl+Shift+M 选择括号内的内容（按住-继续选择父括号）
  • Ctrl+Shift+↑可以移动此行代码，与上行互换
  • Ctrl+Shift+↓可以移动此行代码，与下行互换
  • Ctrl+鼠标左键 可以同时选择要编辑的多处文本

linux-svn

  1. 将服务器上的文件checkout到本地目录

    svn checkout path 简写 svn co path

  2. 添加文件到版本库

    svn add file

  3. 提交改动到版本库

    svn commit -m "commit" 简写 svn ci -m "commit"

  4. 加锁/解锁

    svn lock -m "commit"
    ‘svn unlock path’

  5. 更新代码到某版本

    1. svn update 将当前目录以及子目录下的所有文件更新到最新版本
    2. svn update -r 200 test.php 将test.php更新到版本200
    3. svn update test.php更新test.php到最新版本

  6. 查看文件/目录状态

    1. svn status path 目录下的文件和子目录的状态,正常状态不显示
      • ?: 不在svn的控制中
      • M:内容被修改
      • C:发生冲突
      • A:预订加入到版本库
      • K:被锁定
    2. svn status -v path

  7. 删除文件

    svn delete file

  8. 查看日志

    svn log path

  9. 查看文件详细信息

    svn info path

  10. 比较差异

    1. svn diff path 将修改的文件与基础版本比较
    2. svn diff -r m:n path 版本m和版本n比较

  11. 版本间合并

    svn merge -r m:n test.php 比较版本m和n的test.php

  12. 帮助

    svn help

  13. 版本库下的文件和目录列表

    svn list path 目录path下的属于版本库的文件和目录

  14. 创建版本库控制下的新目录

    svn mkdir path

  15. 恢复本地修改

    svn revert 恢复原始未修改的工作副本文件

  16. 代码库的URL修改

    1. svn switch path 更新工作副本,映射到一个新的URL(会将服务器上的文件与本地文件合并)
    2. switch -relocate FROM TO path 修改工作副本的URL元数据,以反应单纯的URL上的改变.当仓库的根URL变动,但是工作副本仍旧对映到同一仓库的同一目录时使用

  17. 解决冲突

    svn resolved path 移除工作副本的目录或文件的冲突状态(只是移除冲突的相关文件,然后让path可以再次提交)

  18. 输出指定文件或URL的内容

    • svn cat 目标[@版本] 如果指定了版本.将从指定的版本开始查找
    • svn cat -r PREV filename > filename PREV是上一版本,也可以写具体版本号

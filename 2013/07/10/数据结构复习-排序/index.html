1. Insertion sort

example:
T = (13, 6, 3, 31, 9, 27, 5, 11)

  • Analysis
    The number of comparisions and assignments of entries depend on the input.
    the best case: n comparisons, 0 assignments
    the worst case: (n -1) + (n - 2) + … + 1 comparisions, (n + 1) + (n - 1 + 1) + … + 3 assignments

  • contiguous implementation

    disadvantage: the time of data movement

  • linked implementation

    advantage: with no movement of data

2. Selection sort

问题: 选择排序是否稳定,交换次数

scanf the unsorted list and find the largest, then exchange this entry with the entry in the last positon.

examples:
T = (21, 25, 49, 25<sup>2</sup>, 16, 08)

时间复杂度:移动少,比较多
空间复杂度:只用1个缓冲单元

3. Shell sort

Insertion sort is inefficient in moving entries only one position at a time. Shell sort can reduce the number of movements.

  • 基本思想: 先将整个待排记录序列分割成若干子序列,分别进行直接插入排序，待整个序列中的记录“基本有序”时，再对全体记录进行一次直接插入排序。
  • 技巧: 子序列的构成不是简单地“逐段分割”，而是将相隔某个增量dk的记录组成一个子序列,让增量dk逐趟缩短(例如依次取5,3,1)，直到dk＝1为止。

  • 优点: 让关键字值小的元素能很快前移，且序列若基本有序时，再用直接插入排序处理，时间效率会高很多。

    examples:
    T = (49, 38, 65, 97, 76, 13, 27, 49', 55, 04)
    dk = 5时: 49-13, 38-27, 65-49', 97-55, 76-04
    dk=5: 13, 27, 49<sup>2</sup>, 55

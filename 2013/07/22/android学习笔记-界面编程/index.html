Android学习笔记-界面编程

说明: 内容取自中山大学软件学院刘宁老师的SE-806课程的课件

view,视图组件

view负责:

  • 说明布局Layout
  • 创建菜单Menus
  • 创建对话Dialogs
  • 处理用户UI事件
  • 用户提示
  • 确定风格
  • 定制组件
  • 数据与界面绑定


View类是最基本的一个UI类,基本上所有的高级UI组件都是继承于View类而实现

一个视图在屏幕占据一块矩形区域,负责渲染这块矩形区域,也可以处理此区域发生的事件,设置是否可见,是否可以获取焦点。
类型:

  • TextView,文本框
  • Button, 按钮
  • List, 列表
  • EditText, 编辑框
  • RadioButton, 单选框
  • CheckBox, 检查框

UI布局

ViewGroup通过各种Layout控制所属的view的显示位置

Layout(布局组件)类型:

  • LinearLayout 线性布局
    • 垂直: android:orientation=”vertical”
    • 水平: android:orientation=”horizontal”
  • FrameLayout
    • 屏幕上故意保留的空白空间
    • 所有子元素都钉到屏幕的左上角,不能为子元素指定位置
  • TableLayout
    • 子元素放到行与列中
    • 不显示边界线
    • 单元格不能跨行
  • AbsoluteLayout 决定定位
    • 指明子元素确切的屏幕坐标(x,y), (0,0)代表左上
    • 允许元素重叠,但不推荐
    • 一般不使用Absolute,除非必要,因为它相当严格且不同设备兼容不好
  • RelativeLayout 相对定位
    • 子元素相对其他元素(通过ID指定)或相对父元素的位置
    • 有丰富的排列属性
      • Layout above
      • Layout below
      • Layout to left of
      • …
    • 尽量少用,排列属性间容易冲突
  • GridView
  • ListLayout
  • RadioGroup
  • …

font

  • android.graphics.Typeface:
    • Typeface.create(Typeface family, int style)
      • 创建一个混合型新的字体:有20种搭配(4种字体5种风格)
    • Typeface.setTypeface (Typeface tf, int style)
      • 设置一个混合型字体:有20种搭配(4种字体5种风格)
    • Typeface.setTypeface(Typeface tf)
      • 设置一个只有Typeface 风格的字体:有五种形式
  • 字体
    • BOLD、BOLD_ITALIC、ITALIC、NORMAL
  • 风格
    • DEFAULT、DEFAULT_BOLD、MONOSPACE(单间隔字体)、SANS_SERIF(无衬线字体)、SERIF(衬线字体)

Menu 菜单

  • Options Menu 主菜单: 通过按下home键来显示,包含两组菜单项:
    • Icon Menu 图标菜单
      用户按下菜单键时最初出现在屏幕下方的item集合,支持最多6个item
    • Expanded Menu 扩展菜单
      多于6个的item通过’更多’菜单显示,由6个以及其他选项菜单组成
    • 实现
      • options menu通过activity的onCreateOptionsMenu来生成
      • onCreateOptionsMenu只会在menu第一次生成时调用,任何想改变options menu的想法只能在onPrepareOptionsMenu来实现,这个函数会在menu显示前调用
      • onOptionsItemSelected用来处理选中的菜单项

  • Context Menu 上下文菜单(右键菜单): 浮动菜单列表,通常长时间按在一个视图上时出现

    • 实现
      • context menu是跟某个具体的view绑定在一起,在activity中用registerForContextMenu来为某个view注册context menu。context menu在显示前都会调用onCreateContextMenu来生成menu。onContextItemSelected用来处理选中的菜单项

  • Sub Menu 子菜单: 浮动菜单列表,通过在Options Menu | Context Menu选择菜单项显露出来,不支持嵌套子菜单.

android还提供了对菜单项进行分组的功能,可以把相似功能的菜单项分成同一个组,这样可以通过调用setGroupCheckable,setGroupEnabled,setGroupVisible来设置菜单属性,而无须单独设置。

进程优先级顺序

  • 前台进程(Active Process):
    • 前台Activity、包含正在运行的广播接收器、正在运行的服务
  • 可视进程(Visible Process):
    • 包含一个可视化的Activity
  • 服务进程(Started Service Process):
    • 包含一个被开启的服务(处理服务,不是直接可视,例如媒体播放器,网络上传、下载)
  • 后台进程(Background Process)
    • 包含一个不可视的Activity(可以在任意时刻杀死该进程来回收内存)
  • 空进程(Empty Process):
    • 没有持有任何应用程序组件

Activity

主要特征
  • 包括UI,以及与user的互动
  • 可以放button、list、picture、text等组件
  • UI可动态调整 (增加、减少、换位置)
  • 利用intent跳转到其他activity
  • 超过5秒,会出现ANR (Android is Not Responding)
  • Activity是Android程序与用户交互的窗口,可以看成网站的页面。
生命周期
  • Active: 当一个Activity在屏幕的最上层时(堆栈的最顶端),它就是属于active或者running的状态
  • Paused: 若Activity 失去焦点(focus)但还看得到它的画面(比如:一个新的Activity 画面并不是全屏幕或者它是一个半透明情况),那失去焦点的Activity 处在paused 状态。这个失去焦点的Activity还是完全活着,并没有消失。(活着的意思是指,Activity 自己本身所有的状态及数据都还是存在的,也跟窗口管理程序window manager 保持联系),像这种paused 的Activity,会在一种情况下消失,那就是当系统的内存不够用之时,系统会自动判断,将不重要的Activity 移除。
  • Stop: 若一个Activity 被其它的Activity 完全的遮盖住时,它仍然保有全部的状态及数据,但因为它已不再被使用者看见,所以它的画面是被隐藏起来的(画面不需要更新),当系统内存不足时,这种stop状态的Activity 时最先被系统考虑拿下来释放内存的。
  • Finish: 当一个Activity 处于pause 或stop 的状态时,系统可以要求Activity 结束(finish)或直接移除(kill)它。当它需要再度呈现在使用者面前时,它必须要能完整的重新启动及回复先前的状态。

linux开机自动挂载windows硬盘分区

各大发行版常用的集成桌面环境（Gnome/KDE等）都会在开机启动时自动发现本地磁盘分区, 并在用户试图进入时挂载该分区. 对于一台装有 Linux/Window7 双系统的电脑, Linux启动后会在文件系统（File System）里显示已发现的 Windows 磁盘分区, 我们进入其中的一个分区, 会提示输入 root 口令, 口令正确时, Gnome 会在 /media 下挂载这个分区, 并以该分区的 UUID 号为挂载目录名.

而当我们基于各种原因不使用庞杂/臃肿的桌面环境时, 转而使用 轻量级窗口管理器（Awesome） + 轻量级文件管理器 （Thunar, ranger）等时, 这种自定义的桌面环境并不支持本地磁盘的开机自动挂载.

用 /etc/fstab 自动挂载本地分区

/etc/fstab 文件告诉 Linux 如何挂载你的电脑中的磁盘分区和其他存储设备, 并且指明它们应该挂载到什么位置. 这个文件中列出的设备会在开机时自动的挂载到相应的挂载点去.

典型的 fstab 文件是一个包含了 6 列、若干行的文本文件, 以 root 权限进行编辑, 我电脑中默认的 fstab 文件内容:

第1列: 设备的UUID, UUID可以换成类似 /dev/sdax 的设备文件名

第2列: 挂载点, 挂载点目录必须在挂载前建立

第3列: 文件系统类型, Linux常用的是 ext2/3/4 , Windows 常用的是 ntfs-3g 和 vfat, 或者使用 auto 关键字让系统自动检测类型.

第4列: 挂载选项, 与mount命令中用到的相同.

defaults 代表的是 rw, suid, dev, exec, auto, nouser, async.

rw 是可读可写, ro 是只可读；

exec/noexec 挂载分区中的文件是否是可执行的；

auto/noauto 表示设备是否在开机时自动挂载；

nouser/user 当该选项为nouser时, 只有root可以挂载该分区, 当其为user时, 普通用户也可以挂载；

async/sync 是否打开同步写入/读出分区内容, async表示否.

一般的, 挂载 Windows分区, 我们采用下面的选项：

defaults, user, locale=zh_CN.utf8

表示采用 defaults 默认的选项, 但改nouser为user, 并指定语言编码以显示中文字符.
注:如果不指定uid和gid, 挂载的ntfs分区的所有者和所属组都是root, 且无法使用chown和chmod改变属性.

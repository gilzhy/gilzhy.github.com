Linux命令

Linux下常用命令:查找文件,vim查找替换,解压/压缩,历史命令,进程管理等

  • 常用查找文件:
    • 当前目录下文件名包含关键字的文件: ll | grep xx , find . -name '*xx*'
    • 当前目录下文件内容包含关键字的文件
      • grep "string" [file]: 输出每个文件中包含”str”的行,并高亮”str”
      • grep -o "str" [file]: 只输出包含”str”的文件
      • find .|xargs grep -ri "str" 输出每个文件中包含”str”的行,无高亮
      • find .|xargs grep -ri "str" -l: 只输出包含”str”的文件
    • 列出当前目录里最大的10个文件: du -s * | sort -n | tail

  1. find [指定目录] [指定条件] [指定动作]

    • 指定目录： 所要搜索的目录及其所有子目录。默认为当前目录。
    • 指定条件： 所要搜索的文件的特征。默认不过滤任何文件

    • 指定动作： 对搜索结果进行特定的处理.默认无处理,全部显示

    • find . -name ‘my*’
      搜索当前目录所有文件名以my开头的文件,显示文件名

    • find . -name ‘my*’ -ls
      搜索当前目录中所有文件名以my开头的文件，显示详细信息。

    • find . -mmin -10
      搜索当前目录中，所有过去10分钟中更新过的所有文件和目录。

    • find . -type f -mmin -10
      搜索当前目录中，所有过去10分钟中更新过的普通文件。

  2. locate

    locate命令其实是“find -name”的另一种写法，但是要比后者快得多

    原因在于它不搜索具体目录，而是搜索一个数据库（/var/lib/locatedb），这个数据库中含有本地所有文件信息。

    Linux系统自动创建这个数据库，并且每天自动更新一次，所以使用locate命令查不到最新变动过的文件。

    为了避免这种情况，可以在使用locate之前，先使用updatedb命令，手动更新数据库。

    • locate ~/xx
      搜索用户主目录下，所有以xx开头的文件。
    • locate -i ~/m
      搜索用户主目录下，所有以xx开头的文件，忽略大小写。

  3. whereis

    whereis命令只能用于程序名的搜索，而且只搜索二进制文件（参数-b）、man说明文件（参数-m）和源代码文件（参数-s）。如果省略参数，则返回所有信息。

  1. which

    which在PATH变量指定的路径中，搜索某个系统命令的位置，并且返回第一个搜索结果

    使用which命令可以查看某个系统命令是否存在，以及执行的到底是哪一个位置的命令

  1. type

    type区分命令是由shell自带的, 还是由shell外部的独立二进制文件提供的。

    如果一个命令是外部命令，那么使用-p参数，会显示该命令的路径，相当于which命令。

  • root相关:

    • 设置root密码: sudo passwd root
    • 转为root: su

  • vim相关

    • 替换: :%s/lala/haha/gc (将lala替换为haha,c表示询问,g表示批量)
    • 查找匹配数: %s/xxx//gn

    • 编辑多个文件:

      • vim a.txt b.txt c.txt

        • :n 跳至下一个文件，:e#回到刚才编辑的文件，:ea.txt 编辑文件a.txt
        • :ls 查看当前打开的文件
        • :b1 打开n缓存区的文件，n为ls查看中文件的编号

      • 使用 sp 或者 vs 命令，对窗口进行分割

        • :vs a.txt 横
        • :sp a.txt 竖
        • ctrl+ww 在窗口间进行切换

  • 解压/压缩

    • gzip file / gunzip file.gz 压缩文件
    • tar -zcvf dir.tar.gz dir / tar -zxvf dir.tar.gz 打包目录
      • -z 打包同时压缩
      • -f 指定压缩后的明文名
      • -v 显示详细信息
      • -c 打包
      • -x 解包
    • zip file.zip file / unzip file.zip
    • zip -r dir.zip dir / unzip dir.zip
      • bzip2 -k file / bunzip2 -k file.bz2 -k表示保留原文件

  • apt

    • apt-get(man apt-get)

      • apt-get update: 更新源(/etc/apt/sources.list记录)
      • apt-get upgrade: 更新系统中所有已经安装的软件至其最新版
      • apt-get install: 安装/更新
      • apt-get remove: 卸载
      • apt-get purged: 清除（在卸载的基础上，删除配置）
      • apt-get download: 下载安装包至当前目录
      • apt-get autoremove: 一些软件安装时必须附带着安装一些其他软件，若卸载了软件，这些附带的软件就不再被需要时，autoremove自动卸载这些

    • apt-cache(man apt-cache)

      • apt-cache showpkg pkg(s): 显示安装包的信息
      • apt-cache stats: 显示统计信息（软件包名称总数、全部软件包结构等）
      • apt-cache showsrc pkg(s): 显示pkg(s)的源地址
      • apt-cache search regex [ regex ... ]: 查找相应软件版本
      • apt-cache show pkg(s): 显示软件详细的信息，同dpkg —print-avail
      • apt-cache depends pkg(s): 显示有哪些安装包依赖于pkg(s)
      • apt-cache rdepends pkg(s): 与depends相反

  • 链接

    • 软链接: ln -s [源文件] [目标文件]
    • 硬链接: ln [源文件] [目标文件]

  • 开关机

    • 关机: shutdown -h now 或init 0
    • 重启: reboot 或init 6

  • 命令别名

    • 设置: alias ll="ll -;"
    • 查看: alias

  • 重定向

    • 输出重定向： ls -l ./tmp > ./tmp.msg 或换成>>
    • 输入重定向: wall < ./temp

  • 命令连接符

    • ; 各命令顺序执行
    • && 只有前面的命令成功执行后,后面的命令才被执行
    • || 只有前面的命令执行失败后,后面的命令才被执行

  • 实用工具

    • 简单服务器: python -m SimpleHTTPServer
    • ascii码表: man ascii

  • 历史命令

    • 以root的身份执行上一条命令: sudo !!
      使用!xx来重复最近的一次命令，比如，你以前输入过vi /where/the/file/is, 下次用 !vi 重得上次最近一次的vi命令
    • 上一个命令的最后一个字符串: !$
    • 查看历史命令: <ctrl>+<R>
    • 把上次命令行的参数给重复出来: ALT+. or <ESC> .
    • 替换前一条命令里的部分字符串: ^old^new (也可以使用 !!:gs/old/new）

  • 创建一个空文件: touch file.txt 或 > file.txt

  • 进程相关

    • 显示进程树: pstree -p
    • 显示包含关键字的进程: ‘ps -ef | grep’
    • 列出头十个最耗内存的进程: ps aux | sort -nk +4 | tail
